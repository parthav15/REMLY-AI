1. The Core Wedge

Why push notifications are dead, and why a phone call works.

We are living in an era of notification blindness. The average smartphone user swipes away 50+ push notifications a day without even reading them.

A phone call creates unavoidable, high-stakes friction. It forces you to stop what you are doing and answer. By leveraging Retell AI natively, we skip the robotic "Press 1 to Snooze" era entirely. The user answers the phone and hears a natural human voice. If they are busy, they simply say, "I'm driving, call me back in 15 minutes," and the AI reschedules it seamlessly.

2. Problem & Personas

The Neurodivergent

Pain: Severe ADHD causes hyper-focus or time blindness. Standard calendars don't break the cycle.

Use Case: "Call me at 3 PM and don't hang up until I confirm I have stopped playing video games and started my homework."

The High-Stakes Pro

Pain: Missing one specific deadline costs money. Cannot risk swiping away an alert.

Use Case: "Call me at 9:55 AM to buy the pre-sale tickets." or "Call me to jump on the investor Zoom."

The Caregiver

Pain: Elderly parents forget to take medication but don't check SMS. They *always* answer the phone.

Use Case: Adult child schedules a daily 8 AM AI call to their mother: "Hi Mom, time to take your blood pressure pill."

5. Technology Stack

Client: Expo + React Native

  • Why: Build once, deploy to iOS and Android. Integrates easily with native contacts (for vCard download) and Apple/Google Pay.

Backend: Django + DRF

  • Why: Django provides a persistent, rock-solid API. The built-in Django Admin instantly solves your V1 admin panel needs. No cold starts.

Database: PostgreSQL

  • Why: Row-level locking (select_for_update) is mandatory when using Cron to ensure the script doesn't double-call users. Postgres handles this perfectly.

Scheduler: OS Cron

  • Why: Simplest possible architecture. No Redis needed. A simple script runs every 60 seconds, finds all due reminders, and pushes them to Retell.

6. The Cron Scheduler Logic

How a reminder gets triggered exactly on time without Celery.

User Saves Reminder → DB: Insert Status=Pending → Cron runs every 1 minute → DB: Select where time <= now() → Django: Retell.createCall()
CRITICAL CRON GOTCHA: Avoiding Double Calls
Your Django management command MUST lock the database rows while processing, otherwise two overlapping cron jobs might call the same user twice. 
# management/commands/run_reminders.py

from django.core.management.base import BaseCommand
from django.utils import timezone
from django.db import transaction
from myapp.models import Reminder

class Command(BaseCommand):
    def handle(self, *args, **kwargs):
        now = timezone.now()
        
        with transaction.atomic():
            # select_for_update() locks these rows until the transaction finishes
            due_reminders = Reminder.objects.select_for_update().filter(
                status='pending',
                scheduled_at__lte=now
            )
            
            for reminder in due_reminders:
                # 1. Trigger Retell API
                trigger_retell_call(reminder)
                
                # 2. Update status so next cron ignores it
                reminder.status = 'triggered'
                reminder.save()

7. Retell AI Integration & Conversational Snooze

Because we are using Cron, the snooze logic is brilliantly simple. We don't need to mess with queues. We just push the database timestamp into the future, and the Cron script will naturally pick it up again when the time comes.

How Conversational Snooze Works

  1. User says: "Hey, I'm stuck in traffic, call me back in 20 minutes."
  2. Retell's LLM detects this intent and triggers the Custom Tool named `snooze_reminder` with the argument `{"minutes": 20}`.
  3. Retell fires an HTTP POST to your Django URL `/api/v1/webhooks/retell/tool/`.
  4. Django finds the reminder in the DB and updates it:
    reminder.scheduled_at = now() + timedelta(minutes=20)
    reminder.status = 'pending'.
  5. Django responds to Retell with `{"status": "success"}`.
  6. Retell's voice says: "Got it, I'll call you back in 20 minutes. Drive safe!" and hangs up.
  7. 20 minutes later, the Cron job sweeps the DB, finds the reminder is due again, and calls them.

11. The Truecaller / Spam Problem

If you buy a phone number in Retell and call someone 4 times a day, telecom carriers will mark your number as "Spam Likely" within a week.

The Fix (Expo App Onboarding Flow)

  1. Buy a single, dedicated phone number inside the Retell dashboard.
  2. During Expo App onboarding, prompt the user to "Save Assistant to Contacts".
  3. Use the expo-contacts package to programmatically save your Retell number as "Remly AI Assistant" on their device.
  4. Why this works: iOS and Android explicitly bypass "Silence Unknown Callers" and carrier spam filters if the number exists in the user's local contact book.
